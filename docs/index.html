03 — Architecture

프로젝트 구조

Trading_Bot/ ├── main.py # 실행 진입점 — 전체 파이프라인 오케스트레이션 ├── requirements.txt # 의존성 패키지 (버전 고정) ├── .env # 인증 정보 (Git 제외) ├── .gitignore ├── src/ │ ├── portfolio_config.py # 종목 구성 · 노션 페이지 ID │ ├── data_fetcher.py # yfinance로 3년 OHLCV 수집 │ ├── analyzer.py # 현재가 · 52주 범위 · 수익률 · RSI · MA │ ├── predictor.py # XGBoost(1M) + Prophet(3·6·12M) 앙상블 │ ├── notion_writer.py # 노션 리포트 페이지 자동 생성 │ └── notifier.py # Gmail SMTP 경량 알림 발송 ├── results/ # 차트 저장 (Git 제외) ├── logs/ # 실행 로그 (Git 제외) └── docs/ └── index.html # 이 문서

analyzer.py

  • 현재가 · 52주 고가/저가 · 위치(%)
  • YTD · 1M · 3M · 6M · 1Y 과거 수익률
  • RSI (14일) · 과매수/과매도 판정
  • MA 20/60/200 · 추세 신호
  • 연율화 변동성 · 배당률

notion_writer.py

  • 💼 투자 포트폴리오 하위에 페이지 자동 생성
  • 당일 리포트 중복 시 아카이브 후 재생성
  • Rate Limit 대응 (0.4초 간격 청크 전송)
  • 카테고리별 포트폴리오 요약 섹션
  • 종목별 과거 수익률 + 예측가 상세 섹션
04 — Prediction Model

예측 모델 구성

단기 — 1개월 후

XGBoost + 기술적 지표

RSI · MA 비율 · 5/20일 수익률 · 변동성을 특성으로 사용. 분위수 회귀(10th / 50th / 90th percentile)로 80% 예측 구간 산출. 지표 조합 패턴(RSI 고점 + MA 하락 등)을 비선형 학습.

중·장기 — 3 / 6 / 12개월 후

Prophet (Meta)

트렌드 + 주간 계절성 + 연간 계절성 분해 학습. changepoint_prior_scale=0.05로 안정형 자산에 적합한 보수적 변화점 설정. 12개월 예측에는 배당률을 반영한 총수익률(Total Return) 추가 산출.

구간 모델 예측 구간 배당 투자 활용처
1개월 후XGBoost80% (분위수 회귀)이번 달 DCA 매수 타이밍 참고
3개월 후Prophet80% (신뢰 구간)다음 분기 배당 전후 포지션 점검
6개월 후Prophet80% (신뢰 구간)반기 리밸런싱 기준 참고
12개월 후Prophet + 배당80% (신뢰 구간)연간 자산 성장 목표 대비 점검
주가 예측은 참고용입니다. 단기(1개월) 예측은 노이즈가 크며, 중·장기(6·12개월)가 이 전략에서 더 실용적입니다.
05 — Setup

설치 및 실행

1. 의존성 설치

pip install -r requirements.txt

2. 환경변수 설정 (.env)

EMAIL_ADDRESS=your_email@gmail.com
EMAIL_PASSWORD=your_google_app_password
TARGET_EMAIL=your_email@gmail.com
NOTION_TOKEN=your_notion_integration_token

3. Notion 통합 연결

  • notion.so/my-integrations 에서 Internal Integration 생성
  • 💼 투자 포트폴리오 페이지 → ··· → Connections → 통합 선택
  • 하위 페이지에 자동 권한 상속

4. 실행

cd "Trading_Bot"
python main.py

06 — Automation

자동화 — Windows 작업 스케줄러

매일 오전 7:00 KST에 자동 실행됩니다. 미국 장 마감(오전 5:00 KST) 이후 2시간 버퍼로 당일 종가가 완전히 반영된 후 분석합니다.

등록 명령 (관리자 PowerShell)

$action = New-ScheduledTaskAction `
    -Execute "python.exe" `
    -Argument "main.py" `
    -WorkingDirectory "C:\...\Trading_Bot"

$trigger = New-ScheduledTaskTrigger -Daily -At 7:00AM

Register-ScheduledTask -TaskName "IVV_Trading_Bot" `
    -Action $action -Trigger $trigger -RunLevel Highest -Force

결과물

  • 노션: 날짜별 리포트 페이지 자동 생성
  • 이메일: 과매수/과매도 신호 + 노션 링크
  • 로컬: logs/run_log.txt 실행 기록

리스크 관리

  • yfinance 버전 고정 — API 변경 대응
